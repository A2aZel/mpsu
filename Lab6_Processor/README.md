# ЛР6. Процессор RISC-V (тракт данных + LSU + прерывания)

Это объединённый результат трёх работ учебного плана:
ЛР07 (тракт данных), ЛР09 (интеграция LSU) и ЛР11 (интеграция прерываний).
Получается полноценное однотактное ядро RISC-V (RV32I + Zicsr + `mret`)
с памятью инструкций, LSU и памятью данных.

## Мои файлы (решение)
- `processor_core.sv` — ядро (PC, декодер, РФ, АЛУ, 5 констант, LSU-интерфейс,
  CSR и контроллер прерываний, мультиплексоры `a_sel`/`b_sel`/`wb_sel`,
  логика перехода, `jalr` с обнулением младшего бита).
- `processor_system.sv` — система: `core` + `imem` + `lsu` + `data_mem`
  (плюс провода `irq_req`/`irq_ret`).
- `data_mem.sv` — память данных (синхронное чтение, побайтовая запись).
- `lsu.sv` — блок загрузки/сохранения (byte enable, расширение, `stall`).
- `csr_controller.sv` — регистры контроля и статуса (mie/mtvec/mscratch/mepc/mcause).
- `interrupt_controller.sv` — контроллер прерываний (`irq`/`irq_ret`).

## Вспомогательные файлы (зависимости из ЛР1–ЛР5)
- `decoder.sv`, `decoder_pkg.sv`, `csr_pkg.sv`, `alu.sv`, `alu_opcodes_pkg.sv`,
  `register_file.sv`, `instr_mem.sv`, `memory_pkg.sv`,
  `fulladder.sv`, `fulladder4.sv`, `fulladder32.sv`.
- `program.mem` — тестовая программа (из ЛР11/ЛР10, демонстрирует прерывание).
- `lab_11.tb_processor_system.sv` — тестбенч системы.

## Как загрузить в Vivado
1. Создать пустой RTL-проект.
2. В `Design Sources` добавить все `.sv` из папки и файл `program.mem`.
3. В `Simulation Sources` добавить `lab_11.tb_processor_system.sv`
   (верхний модуль симуляции — `lab_11_tb_processor_system`).
4. Запустить симуляцию. Тест без автоматического вердикта: нужно проверить
   по временной диаграмме (`core.pc`, `instr`, регистры `core.i_rf.rf_mem`,
   обработку прерывания) корректность выполнения программы.
   Тест завершается, когда ядро отрабатывает прерывание и выставляет `irq_ret`.

> Имена сущностей соответствуют требованиям окружения: экземпляр ядра — `core`,
> память инструкций — `imem`, провода прерывания — `irq_req` и `irq_ret`.
