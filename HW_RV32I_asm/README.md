# ДЗ. Программа на ассемблере RISC-V (RV32I) — вариант 12

Файл: `hw_variant12.txt` — программа на ассемблере RISC-V.

## Задание
Вариант 12 из ЛР4: сформировать число, состоящее из **чётных двоичных разрядов**
входного значения (биты 0, 2, 4, …), упакованных подряд: `result[k] = input[2*k]`.

## Данные в ОЗУ
- `input`  — слово с исходными данными (в Venus/Ripes адрес `0x10010000`).
- `result` — слово с результатом (адрес `0x10010004`).
- Результат также дублируется в `x10` и печатается в консоль.

Чтобы проверить на других данных, измените значение `input` в секции `.data`.

## Как запустить (Venus)
Открыть [Venus](https://venus.cs61c.org/) → вкладка **Editor**, вставить
содержимое `hw_variant12.txt` → вкладка **Simulator** → **Assemble & Simulate
from Editor** → **Run**. В консоли снизу напечатается результат, программа
завершится **без ошибок** (корректный выход через `ecall`).

> Если уже был вставлен старый код — очистите редактор и вставьте заново.
> Ошибка `Ran for more than the max allowed steps` раньше возникала из-за
> финального бесконечного цикла `j end`; теперь программа завершается через
> системный вызов `exit`.

> **Ripes/RARS:** там номер системного вызова кладётся в `a7`, а не в `a0`.
> Замените `li a0, 1` → `li a7, 1` и `li a0, 10` → `li a7, 10`
> (либо просто оставьте в конце `end: j end` и остановите симуляцию вручную).

## Проверка
Для примера `input = 0x000003B8` (двоично `011_1011_1000`) программа печатает и
сохраняет в `result` значение `0b01_0100` = **20** (`0x14`).
