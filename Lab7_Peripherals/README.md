# ЛР7. Периферийные устройства (Переключатели + Светодиоды)

Завершение процессорной системы: добавлена системная шина, делитель частоты
и два контроллера периферии — переключателей (адрес `0x01`) и светодиодов (`0x02`).
Теперь `processor_system` — модуль верхнего уровня, работающий напрямую с ПЛИС.

## Мои файлы (решение)
- `sw_sb_ctrl.sv` — контроллер переключателей: чтение `sw_i` по адресу `0x00`,
  прерывание при изменении переключателей (удерживается до `interrupt_return_i`).
- `led_sb_ctrl.sv` — контроллер светодиодов: регистр `led_val` (`0x00`),
  режим "моргания" `led_mode` (`0x04`, счётчик до 2·10⁷), сброс по `0x24`.
- `processor_system.sv` — финальная система: `sys_clk_rst_gen` (sysclk/rst),
  ядро `core`, `imem`, `lsu`, `data_mem`, one-hot дешифратор `addr[31:24]`,
  контроллеры `sw_ctrl`/`led_ctrl`, мультиплексор `read_data`, линии прерывания.

## Вспомогательные файлы
- `peripheral_pkg.sv` — старшие адреса периферии.
- `sys_clk_rst_gen.sv` — делитель частоты (использует примитив `BUFG`).
- `nexys_a7_100t.xdc` — файл ограничений (выводы ПЛИС Nexys A7).
- Все модули процессора из ЛР6 (`processor_core.sv`, `lsu.sv`, `data_mem.sv`,
  `csr_controller.sv`, `interrupt_controller.sv`, декодер, АЛУ, РФ, память, пакеты).
- `program.mem` — прошивка `sw_led` (опрос прерывания от переключателей →
  вывод на светодиоды), исходник — `sw_led.S`.
- `lab_13.tb_processor_system.sv` — тестбенч.

## Как загрузить в Vivado
1. Создать пустой RTL-проект (целевая плата Nexys A7-100T, xc7a100tcsg324-1).
2. В `Design Sources` добавить все `.sv` и `program.mem`.
3. Добавить `nexys_a7_100t.xdc` в `Constraints`.
4. Верхний модуль — `processor_system`.
5. Для симуляции добавить `lab_13.tb_processor_system.sv`
   (верхний модуль симуляции — `lab_13_tb_processor_system`).
   Тест без автоматического вердикта — смотрим по временной диаграмме, что после
   изменения `sw_i` приходит прерывание, процессор читает переключатели и пишет
   значение в контроллер светодиодов.

> [!IMPORTANT]
> Для ускорения симуляции "моргания" можно временно уменьшить `HALF_PERIOD`
> в `led_sb_ctrl.sv` (например в 1000 раз). Перед генерацией битстрима для платы
> значение нужно вернуть к `10_000_000`, иначе моргание будет слишком быстрым.
